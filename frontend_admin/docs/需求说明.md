# BUPT 智能充电桩调度计费系统管理端需求说明

## 1. 项目范围

本项目当前实现的是管理员客户端，客户端由其他成员负责。管理端使用 Vue 3 + Vite，采用前后端分离方式，后端 API 统一前缀为 `/api/v1`。

管理端重点围绕充电站运营和调度：充电桩监控、队列查看、故障调度、故障恢复、扩展调度、车辆信息维护和运营报表。

## 3. 管理员职责覆盖情况

管理员客户端需求包括：

- 启动/关闭充电桩：已实现。
- 查看所有充电桩状态：已实现。
- 查看各充电桩等候服务车辆信息：已实现。
- 处理充电桩故障并选择调度策略：已实现入口。
- 处理故障恢复后的重新调度：已预留入口。
- 查看报表：已实现页面和字段。
- 用户/车辆信息维护：已实现车辆维护页面。

## 4. 核心业务参数

| 配置项 | 默认值 | 说明 |
| --- | --- | --- |
| 快充桩数量 | 3 | 快充功率 30 度/小时 |
| 慢充桩数量 | 2 | 慢充功率 10 度/小时 |
| 充电桩队列长度 | 5 | 每个充电桩队列最多 5 辆车 |
| 等候区容量 | 10 | 等候区最大车位容量 |
| 快充排队号 | F1、F2... | 对应 `FAST` |
| 慢充排队号 | T1、T2... | 对应 `SLOW` |

## 5. 页面说明

### 5.1 登录页

路径：`/#/login`

功能：

- 管理员登录。
- 本地测试账号：`admin / 123456`。
- 非管理员账号不能进入业务页面。

### 5.2 管理员工作台

路径：`/#/admin`

功能：

- 展示快充桩、慢充桩、等候区容量、队列长度。
- 展示所有充电桩状态。
- 字段包括充电桩编号、类型、状态、是否正常、累计次数、累计时长、累计电量、队列长度、当前任务。
- 支持启动、关闭、模拟故障、查看队列。
- 队列车辆表展示排队号、用户 ID、模式、电池容量、请求电量、排队时长、预计完成时间、状态。

### 5.3 调度中心

路径：`/#/admin/scheduling`

功能：

- 可视化展示等候区车辆。
- 可视化展示充电区 5 个充电桩队列。
- 展示叫号状态：正常开启或故障调度期间暂停。
- 支持故障调度，策略包括 `PRIORITY` 和 `TIME_ORDER`。
- 支持故障恢复调度入口。
- 支持单次调度总充电时长最短入口。
- 支持批量调度总充电时长最短入口。

调度规则：

```text
车辆完成充电所需时长 = 等待时间 + 自己充电时间
等待时间 = 选定充电桩队列中所有车辆完成充电时间之和
自己充电时间 = 请求充电量 / 充电桩功率
```

### 5.4 车辆信息维护

路径：`/#/admin/vehicles`

功能：

- 查询车辆信息。
- 展示用户 ID、用户名、手机号、车牌号、电池容量、当前排队号、状态。
- 支持编辑和新增车辆信息。
- 后端未连接时可使用演示数据查看页面效果。

### 5.5 运营报表

路径：`/#/admin/reports`

功能：

- 支持日、周、月筛选。
- 展示累计充电次数、累计充电时长、累计充电量、累计总费用。
- 表格字段包括时间、充电桩编号、累计次数、累计时长、累计电量、充电费用、服务费用、总费用。

## 7. 数据兜底说明

为了方便在后端未完成时检查页面效果，前端提供了演示数据：

- 充电桩状态演示数据。
- 等候区车辆演示数据。
- 各充电桩队列演示数据。
- 车辆维护演示数据。
- 运营报表演示数据。

后端接口可用后，页面会优先使用接口返回数据；接口失败时展示演示数据并提示“接口暂未连接”。

## 8. 非功能需求

- 统一通过 `Authorization: Bearer {token}` 传递登录凭证。
- 统一处理响应结构 `{ code, message, data }`。
- `code = 200` 表示成功，其余状态按错误提示处理。
- 时间格式使用 `yyyy-MM-dd HH:mm:ss`。
- 金额展示保留 2 位小数。
- 当前前端只作为管理员客户端交付，普通用户页面不作为交付重点。

# 管理端后端联调接口说明

本文档用于前后端联调。当前前端管理端页面已经完成，后端需要提供以下接口数据和业务逻辑。

## 1. 通用约定

### 1.1 API 前缀

```text
/api/v1
```

### 1.2 Token

登录后，前端会在请求头携带：

```text
Authorization: Bearer {token}
```

### 1.3 统一响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

### 1.4 通用状态枚举

```text
充电模式：FAST, SLOW
充电桩状态：IDLE, CHARGING, FAULT, OFFLINE
请求/车辆状态：WAITING, CHARGING, COMPLETED, CANCELLED, FAULT_STOPPED
调度策略：PRIORITY, TIME_ORDER
```

## 2. 管理员登录

### 接口

```text
POST /api/v1/user/login
```

### 请求

```json
{
  "username": "admin",
  "password": "123456"
}
```

### 响应

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "userId": 1,
    "username": "admin",
    "token": "jwt-token",
    "role": "ADMIN"
  }
}
```

### 说明

前端只允许 `role = ADMIN` 的账号进入管理端。

## 3. 充电桩状态监控

### 接口

```text
GET /api/v1/pile/status
```

### 响应

```json
{
  "code": 200,
  "message": "success",
  "data": [
    {
      "pileId": 1,
      "name": "F1 快充桩",
      "type": "FAST",
      "status": "CHARGING",
      "isWorking": true,
      "power": 30,
      "totalCount": 18,
      "totalDuration": 12.6,
      "totalKwh": 378
    }
  ]
}
```

### 字段说明

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| pileId | number | 充电桩 ID |
| name | string | 充电桩名称 |
| type | string | FAST / SLOW |
| status | string | IDLE / CHARGING / FAULT / OFFLINE |
| isWorking | boolean | 是否正常工作 |
| power | number | 功率，快充 30，慢充 10 |
| totalCount | number | 系统启动后累计充电次数 |
| totalDuration | number | 累计充电总时长，单位小时 |
| totalKwh | number | 累计充电总电量，单位度 |

## 4. 充电桩队列车辆

### 接口

```text
GET /api/v1/pile/queue/{pileId}
```

### 响应

```json
{
  "code": 200,
  "message": "success",
  "data": [
    {
      "queueNumber": "F1",
      "userId": 10001,
      "mode": "FAST",
      "batteryCapacity": 65,
      "requestedKwh": 28,
      "waitingMinutes": 0,
      "estimatedFinishMinutes": 56,
      "status": "CHARGING"
    }
  ]
}
```

### 字段说明

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| queueNumber | string | 排队号，如 F1、T2 |
| userId | number | 用户 ID |
| mode | string | FAST / SLOW |
| batteryCapacity | number | 车辆电池总容量，单位度 |
| requestedKwh | number | 请求充电量，单位度 |
| waitingMinutes | number | 已排队时长，单位分钟 |
| estimatedFinishMinutes | number | 预计完成时间，单位分钟 |
| status | string | WAITING / CHARGING / FAULT_STOPPED |

### 启动接口

```text
POST /api/v1/pile/start/{pileId}
```

### 关闭接口

```text
POST /api/v1/pile/stop/{pileId}
```

### 响应

```json
  "message": "success"

```

### 说明

操作完成后，前端会重新调用 `/pile/status` 刷新状态。

## 6. 故障模拟与故障调度

### 接口

```text
POST /api/v1/fault/simulate/{pileId}
```

### 请求

```json
{
  "schedulePolicy": "PRIORITY"
}
```

### schedulePolicy 可选值

```text
PRIORITY：优先级调度
TIME_ORDER：时间顺序调度
```

### 响应建议

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "faultPileId": 3,
    "schedulePolicy": "PRIORITY",
    "pausedCalling": true,
    "generatedBillId": 8001,
    "affectedVehicles": [
      {
        "queueNumber": "F2",
        "userId": 10002,
        "status": "FAULT_STOPPED"
      }
    ]
  }
}
```

### 后端需要实现的业务

- 故障桩正在充电车辆停止计费。
- 为本次已发生充电过程生成一条详单。
- 暂停等候区叫号。
- 按选择的策略重新调度故障队列车辆。
- 调度完成后恢复等候区叫号。

## 7. 故障恢复调度

### 前端预留接口

```text
POST /api/v1/fault/recover/{pileId}
```

### 请求

```json
{
  "schedulePolicy": "TIME_ORDER"
}
```

### 响应建议

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "recoveredPileId": 3,
    "schedulePolicy": "TIME_ORDER",
    "pausedCalling": true,
    "rescheduled": true
  }
}
```

### 后端需要实现的业务

- 将故障充电桩状态从 `FAULT` 恢复为 `IDLE` 或可工作状态。
- 若其它同类型充电桩中仍有未充电车辆，则暂停等候区叫号。
- 合并其它同类型充电桩未充电车辆。
- 按排队号码先后顺序重新调度。
- 调度完成后恢复等候区叫号。

## 9. 单次调度总充电时长最短

### 前端预留接口

```text
POST /api/v1/schedule/dispatch-once
```

### 请求

```json
{
  "mode": "BY_MIN_TOTAL_TIME"
}
```

### 后端需要实现的业务

- 当充电区出现多个空位时，一次叫多个号。
- 被叫车辆不考虑排队先后顺序。
- 按充电模式分配对应类型充电桩。
- 目标是使本次进入车辆完成充电总时长最短。

### 响应建议

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "dispatchType": "ONCE",
    "assignedVehicles": [
      {
        "queueNumber": "F4",
        "from": "WAITING_AREA",
        "toPileId": 2,
        "estimatedFinishMinutes": 40
      }
    ]
  }
}
```

## 10. 批量调度总充电时长最短

### 前端预留接口

```text
POST /api/v1/schedule/dispatch-batch
```

### 请求

```json
{
  "mode": "BY_MIN_TOTAL_TIME"
}
```

### 后端需要实现的业务

- 只有当车辆数达到全部车位数量时，才开始一批调度。
- 本策略不区分快充和慢充模式。
- 车辆可分配任意类型充电桩。
- 目标是使整批车辆完成充电总时长最短。
- 不考虑修改请求和充电桩故障。

### 响应建议

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "dispatchType": "BATCH",
    "assignedVehicles": [
      {
        "queueNumber": "B1",
        "userId": 10001,
        "toPileId": 1,
        "estimatedFinishMinutes": 48
      }
    ]
  }
}
```

## 12. 运营报表

### 前端预留接口

```text
GET /api/v1/report/list?period=DAY
```

### period 可选值

```text
DAY
WEEK
MONTH
```

### 响应

```json
{
  "code": 200,
  "message": "success",
  "data": [
    {
      "period": "2026-06-07",
      "pileId": 1,
      "totalCount": 18,
      "totalDuration": 12.6,
      "totalKwh": 378,
      "electricityFee": 298.2,
      "serviceFee": 302.4,
      "totalFee": 600.6
    }
  ]
}
```

### 字段说明

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| period | string | 日、周、月对应的时间 |
| pileId | number | 充电桩编号 |
| totalCount | number | 累计充电次数 |
| totalDuration | number | 累计充电时长，单位小时 |
